# Roadmap & Task System

## Structure
```
roadmap/
├── README.md           # Ce fichier — vue d'ensemble
├── backlog.md          # Toutes les idées et features futures
├── current-sprint.md   # Sprint en cours (ce qu'on fait maintenant)
└── done.md             # Features livrées (historique)
```

## Workflow
1. Les idées entrent dans `backlog.md` avec une priorité
2. On déplace dans `current-sprint.md` ce qu'on va faire maintenant
3. Chaque feature est décomposée en tasks atomiques
4. Une fois livrée → déplacée dans `done.md`

## Priorités
- **P0** — Critique, bloquant, à faire immédiatement
- **P1** — Important, à faire dans le sprint courant
- **P2** — Souhaitable, prochain sprint
- **P3** — Nice-to-have, quand on aura le temps
